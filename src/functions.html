Functions

Note that for the context of this guide, functions and methods are analogues

Signature

Arguments

Functions should have a maximum of 6 arguments. If more arguments are needed, consider grouping them into a structure.

Pass by Value

Variables should be passed by value only if there is a need to copy the value into a separate instance. If the variable does not necessarily need to be copied or reference the passed variable but is represented by a primitive type, then you should always pass it by value.

Do not declare arguments passed by value as constant even though modifying arguments in the body of a method should be avoided. This semantic freedom is granted primarily to reduce code bloat.

Pass by Reference

All objects should be passed by const reference by default.

Pass by Pointer

Return Values

Only new variables that differ from the input should be returned.

Error conditions that extend past boolean conditions should return an instance of some error class or enumeration. Avoid returning raw integers

Overloading

By default, do not overload unless there becomes a need to simplify calling the method.

Default Values

Avoid default values. Only set them when it is safe and improves calling convenience. Never supply a default value for the sake of providing a default value. If the goal is to provide a default behaviour, consider writing a wrapper that clearly states it performs default behaviour in its name.
