File Conventions

Header Files

Header files always use the .h extension. After the boilerplate, they must have inclusion guards defined with the following format: TODO FMT _ProjectName_FileName_H. Following that, a project consistent namespace must be declared.

Up to one class may be defined in a header file. An unlimited number of other data types may be defined (struct, enum, union, typedef, etc.).

Avoid including other header files when possible. Do not declare members as pointers to avoid an inclusion.

Avoid implementing methods in the header file unless they are small. 1-2 line set/get methods maybe be squashed to a single line.

Source Files

Source files always use the .cpp extension. After the boilerplate, they must declare a project consistent namespace.

Up to one class may be implemented

Avoid defining data types. Use nested definitions in the header file instead.

Other Files

Directory Format

All code files should reside somewhere under the directory /ProjectRoot/Src/. Subsystems should have their own directory from the parent. For example: /Src/Core/ and /Src/Physics/.

Header and source files dealing with the same class should be in the same directory. For example: /Src/Physics/Body.h and /Src/Physics/Body.cpp
